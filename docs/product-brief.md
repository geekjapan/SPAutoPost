# SPAutoPost Product Brief

## 目的

SPAutoPost は、社内 SharePoint サイトのお知らせ掲示板に、セキュリティ対策情報、脆弱性情報、その対応方法を掲載するための自動化プログラムです。

当面は、脆弱性情報の収集、対策方針の整理、掲示板向け文章の生成、SharePoint への下書きまたは投稿までをこのプログラム内に実装します。

将来的には、クローラー、情報収集、情報整理を別プログラムに分離し、SPAutoPost は正規化済み情報を受け取って、社内掲示板向けの作文、レビュー、投稿、監査記録に責務を絞ることを想定します。

## 想定利用者

- セキュリティ担当者
- CSIRT / SOC / 情報システム部門
- SharePoint お知らせ掲示板の運用担当者
- 社内ユーザー向けに脆弱性対策情報を発信する担当者

## 投稿対象

- 社内 SharePoint サイト
- お知らせ掲示板
- 形式は未確定。SharePoint List item または SharePoint Site Page を候補とする

## 掲載する情報

- 重要な脆弱性情報
- 悪用確認済みまたは悪用可能性の高い脆弱性
- ベンダー公開のセキュリティアップデート
- 社内利用製品に関係する注意喚起
- 一般利用者向けの対応方法
- 管理者向けの対応方法
- 参考リンク、出典、影響範囲、緊急度

## 生成 AI の利用

掲示板向け文章の作成には生成 AI を利用します。

実稼働では、次のようなエンジンに対応する想定です。

- Copilot Studio
- Microsoft Foundry / Azure OpenAI
- その他の LLM API

テスト用途では、次のような人間操作または限定的な検証用エンジンを扱う可能性があります。

- ChatGPT サブスクリプション
- Claude サブスクリプション

ただし、サブスクリプション型のチャット UI は、API 利用、非人間アクセス、自動化、業務データ投入、利用規約、監査性の制約を別途確認する必要があります。そのため、実稼働用 provider とテスト用 provider は明確に分離します。

## 基本方針

1. GitHub の Spec / Milestone / Issue を正本とする。
2. SharePoint への投稿は、初期段階では直接公開ではなく、下書きまたはレビュー前提とする。
3. 生成 AI の出力は、そのまま自動公開しない。少なくとも MVP では人間レビューを挟む。
4. 脆弱性情報は出典を保持し、生成文にも参照元を残す。
5. 収集・整理・作文・投稿を疎結合に設計し、将来のクローラー分離に備える。
6. 投稿処理は冪等性を持たせ、重複投稿を防ぐ。
7. 秘密情報、API キー、認証情報、社内限定情報をログに出さない。
8. 監査可能性を重視し、誰が、何を、どの情報源に基づき、いつ投稿したかを追跡可能にする。

## 非目的

初期段階では、次は対象外とします。

- 社内資産台帳との完全自動照合
- EDR / SIEM / ITSM との本格連携
- 人間レビューなしの完全自動公開
- SNS や外部公開サイトへの投稿
- 脆弱性スキャンそのもの
- 攻撃コード、PoC、悪用手順の生成

## 成功条件

初期 MVP の成功条件は次のとおりです。

- 手動入力または簡易収集した脆弱性情報から、社内掲示板向けの日本語原稿を生成できる。
- 原稿には、概要、影響、対象、対応方法、参考情報、緊急度が含まれる。
- 生成 AI provider を差し替え可能な形で実装できる。
- SharePoint に下書きまたはテスト投稿できる。
- 投稿前レビュー、出典確認、重複投稿防止、監査ログの最小構成がある。
